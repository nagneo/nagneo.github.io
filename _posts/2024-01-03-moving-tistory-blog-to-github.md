블로그를 새 플랫폼으로 옮겼다. 첫번째 글은 어떤 동기로 이사를 결정하게 되었는지, 그리고 선택한 깃허브 블로그 테마에 대한 후기를 공유해보고자 한다.


### 이사 동기

1월은 언제나 새로운 시작을 할 때 좋은 달이라고 생각한다. 그래서 2024년을 맞아 기존 블로그에 이어 새로운 블로그를 만들어보기로 결심했다. 기존 블로그에는 이미 30여개의 글이 있었지만, 새로운 기술 공부 기록을 위해 이사를 결정했다.

이번에 선택한 깃허브 블로그는 티스토리나 네이버 블로그와는 다르게 기술 커뮤니티에서 널리 사용되는 플랫폼이다. 프론트엔드 공부를 더 집중적으로 기록하고 싶었고, 더 목적 지향적인 플랫폼을 선택하려고 했다.

도메인을 사서 개발 블로그를 만들고 싶은 욕망도 있었지만, SEO를 위해서는 정적 웹사이트 생성이 필수적이어서 고려할게 많았다. (코드 스니펫, 에디터, 프레임워크 및 기술 스택 선정 등)

### **깃허브** 블로그

1. **커스텀 가능한 블로그**: Jekyll이라는 정적 사이트 생성기를 활용하여 실제 HTML 파일을 만들어내는 작업이 개발자로서 더 매력적으로 다가왔다. 에디터에서 직접 작업하지 않고 Markdown 파일로 작성하여 블로그를 커스터마이징할 수 있다.

2. **세련되고 다양한 공개 테마**: Jekyll은 이미 다양하고 예쁜 테마들이 많이 존재한다. 기술 커뮤니티에서 널리 사용되는 플랫폼이기 때문에 세련된 테마를 쉽게 찾을 수 있었다.

3. **커밋 로그 기록하기**: 개발자로서 커밋과 기록을 동시에 할 수 있는 깃허브의 장점을 활용하기로 했다. 블로그를 통해 기술적인 내용을 다루면서 동시에 깃허브의 잔디를 심을 수 있다는 점이 매력적이었다.

사실 디자인만 생각한다면 나는 깔끔한 블로그를 선호하기 때문에 velog도 좋은 선택지라고 생각되었지만, 1번 3번의 이유로 최종적으로 github를 선택했다.  


### 테마 선택

> 🌎 세상은 넓고 테마는 많다

Jekyll을 선택했지만, 테마 선택은 쉽지 않았다. 다양한 테마 중에서 내게 맞는 테마를 찾는 데 시간을 많이 소비했다. 
내가 원하는 조건은 다음과 같았다.

1. light, dark 모드 지원 가능
2. 사이드바 형태로 개요 표시
3. 심플한 디자인 (위쪽 tab형태의 navigation)
4. 코드 스니펫 

테마를 선택하는 과정에서 [깃헙 토픽](https://github.com/topics/jekyll-theme), [Jekyll Themes](https://jekyll-themes.com/), [Free Jekyll Themes](https://jekyllthemes.io/free) 등의 사이트를 참고하고 다양한 블로그를 방문하며 내가 원하는 디자인들을 찾았다. 최종적으로는 jekyll-theme-yat라는 테마를 선택했고, 앞으로 더 발전시켜 나가 예정이다.
